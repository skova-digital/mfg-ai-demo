# 検査記録AI デモ — 現場の報告を、そのまま品質データに

> **本デモは架空の業務設定と生成データ（seed 固定の決定論的ダミーデータ）によるものです。実在の企業・製品・人物・実データとは一切関係ありません。**

中小製造業の現場では、検査報告が口頭・メモ・紙帳票のまま埋もれがちだという課題が広く知られています（本デモの題材はこの業界一般の課題設定であり、特定企業の事例ではありません）。このデモは、**現場の雑な報告文や帳票写真を AI がその場で構造化データに変換し、蓄積すると不良傾向がダッシュボードで見えてくる**——という一連の流れを、実際に動く形で示すものです。

## 何が見られるか（3ステップ）

1. **入力**: 現場の口語そのままの報告（例:「MK-1024のロットB、切削上がり50個みたけど2個バリ。うち1個は限度見本ギリ」）や帳票写真を投入
2. **AI 構造化 → 人が確定**: Claude API が品番・ロット・数量・不良分類・判定を抽出。自信のない項目はハイライトされ、**人が確認・修正してから保存**される（AI は下書き係、確定は人——現場に受け入れられるための設計判断です）
3. **ダッシュボード**: 蓄積データから不良率の推移・不良分類の内訳・品番×工程の偏りが見える。同梱のダミーデータには「ある品番のある工程だけ不良率が悪化している」状況を意図的に仕込んであり、開いた瞬間に「気づける」ことを体験できます

## アーキテクチャ

```
[ブラウザ]  入力(テキスト/写真) → 確認・修正 → ダッシュボード
    │ fetch
[Cloudflare Workers]  /api/extract ─→ Claude API (tool use で JSON スキーマ強制)
    │                 /api/records, /api/stats
[Cloudflare D1]  inspection_records（検査記録）
```

- **依存最小**: フロントはビルドなしの素の HTML/CSS/JS、グラフは SVG 手書き、バックエンドは Workers 1ファイル
- **構造化の安定性**: Claude の tool use（structured output）でスキーマ準拠を強制。自由文からの抽出でも型が崩れない
- **デモモード**: 公開版は事前実行済みの AI 変換結果を再生する `EXTRACT_MODE=sample` で動作（API 費用ゼロで恒久公開できる設計）。ライブ変換の様子は下記 GIF と、ローカル実行手順で確認できます

## ローカルで動かす

```bash
npm install -g wrangler   # 未導入の場合
node scripts/gen-dummy.mjs                     # ダミーデータ生成（seed 固定・再現可能）
wrangler d1 execute mfg_ai_demo --local --file=scripts/schema.sql
wrangler d1 execute mfg_ai_demo --local --file=scripts/seed-data.sql
cp .dev.vars.example .dev.vars                 # ライブ変換を試す場合は API キーを設定
wrangler dev                                    # http://localhost:8787
```

`.dev.vars` を設定しない場合も、`EXTRACT_MODE=sample`（既定）でデモ全体が動作します。

## 設計判断メモ

- **なぜ「人が確定」を挟むか**: 検査記録は品質保証の根拠になるデータであり、AI の抽出ミスがそのまま記録になる設計は現場で信頼されない。AI の役割を「転記の下書き」に限定し、確定権を人に残すことで、導入障壁と誤記録リスクを同時に下げる
- **なぜ Workers + D1 か**: サーバー常時起動が不要な業務デモでは、従量課金のエッジ実行が費用面で合理的（本デモの想定費用はゼロ円/月）
- **なぜダミーデータに「物語」を仕込むか**: ダッシュボードの価値は「集計が見える」ことではなく「異常に気づける」こと。それを口で説明せず、開いた瞬間に体験してもらうため
